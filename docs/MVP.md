1.一个web界面的开发平台。核心页面有两个，1个是面向用户的对话框（可以选择对应的github仓库，从新输入或者选择历史）。用于澄清一个需求。当需求确认后，就会开始进入开发。（开发进度可视化面板）点击查看可以看到需求的执行进度。氛围思考阶段和工作阶段以及发布阶段。
2.所有任务和对话背后都有claude code来完成。 只是把命令行的claude code 的对话变成了web界面展示。
3.claude code集成github mcp，可以通过拉取对应代码仓到本地开启开发和迭代。
4.当用户点击开始开发时，应该是创建一个新的CC会话，基于任务信息开始走开发流程。思考阶段会产出一个设计方案，在面板上也能查看该文件。工作阶段，则是开始写代码，可以点击查看正在实现代码的subagent的日志。完了则意味代码分支开发好了。 发布阶段需要人在页面手点，操作完后就自动合并分支到main上。
